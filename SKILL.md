# GK_Proposal — 제안서 PPT 자동 생성 파이프라인

그라운드케이(GroundK)의 제안서 PPT를 처음부터 끝까지 자동 생성하는 스킬입니다.
사용자의 클라이언트 분석 자료를 받아 구조화하고, AI로 브랜드 색상 추출 → 카피 정제 → pptxgenjs 코드 생성 → 시각 QA까지 수행합니다.

## 핵심 원칙

1. **콘텐츠 우선 원칙**: 템플릿보다 논리 구조를 먼저 설계한다
2. **가이드 기반 정밀 매칭**: 표지/간지/마무리는 가이드에 정의된 실측 좌표를 정확히 따른다
3. **대화형 진행**: 정보가 부족하면 사용자에게 물어보고, 단계별 확인을 받으며 진행한다
4. **4분류 체계**: 모든 페이지는 표지/간지/본문-기본/본문-강조/마무리로 분류한다

## 필수 참조 파일

이 스킬에 포함된 가이드 파일입니다. 각 단계에서 필요한 파일을 Read로 로드하세요.
경로는 이 스킬의 base directory 기준 상대 경로입니다.

| 파일 | 경로 | 용도 | 언제 읽는가 |
|------|------|------|-----------|
| **생성프롬프트 가이드** | `references/01_생성프롬프트_가이드.md` | STEP 1~4 워크플로우, 색상 시스템, 헤더 좌표, 코드 생성 가이드 | Phase 3 시작 전 |
| **제작시나리오 가이드** | `references/02_제작시나리오_가이드.md` | 43개 템플릿 설명, 조합 패턴, 품질 체크리스트 | Phase 2 템플릿 매핑 시 |

> **가이드 파일에 모든 디자인 규격(좌표, 폰트, 색상 등)이 포함되어 있습니다.** 별도의 마스터 템플릿 파일(.pptx)은 필요하지 않습니다.

## AI 모듈 스크립트

`scripts/` 디렉토리에 있는 Node.js 모듈들을 순서대로 실행합니다:

| 스크립트 | 역할 |
|---------|------|
| `scripts/ai_brand_analyzer.js` | 로고에서 색상 추출, 6:3:1 팔레트 생성, 그라디언트 배경 생성 |
| `scripts/ai_copy_refiner.js` | Claude API로 표지/간지/마무리/헤더 카피 정제 + 일관성 검증 |
| `scripts/ai_visual_qa.js` | 생성된 PPT를 이미지로 변환 후 Claude Vision으로 품질 분석 |
| `scripts/ai_pipeline.js` | 위 3개 모듈의 통합 오케스트레이터 |

> **환경 요구사항**: `ANTHROPIC_API_KEY` 환경변수 필요. 패키지: pptxgenjs, sharp, @anthropic-ai/sdk, adm-zip

## 전체 파이프라인 (5 Phase)

```
사용자 요청 → Phase 0: 자료 수집/인터뷰
                ↓
           Phase 1: 브랜드 분석 (색상 + 배경)
                ↓
           Phase 2: 콘텐츠 구조화 (논리설계 → 템플릿 매핑)
                ↓
           Phase 3: PPT 코드 생성 (pptxgenjs)
                ↓
           Phase 4: 시각 QA + 수정
                ↓
           최종 .pptx 파일 전달
```

## Phase 0: 자료 수집 및 인터뷰

사용자의 요청을 받으면, 아래 정보를 파악합니다. 이미 제공된 정보는 건너뛰고, 부족한 부분만 질문합니다.

### 필수 수집 항목

```
1. 기본 정보: 사업명, 발주처, 사업기간, 예산규모
2. 배경/현황: 현재 과제, 시장 환경
3. 제안 내용: 솔루션, 핵심 기능, 차별점
4. 실적: 유사 프로젝트, KPI
5. 조직: 투입 인력, 구성
6. 비용/일정: 견적, 마일스톤
7. 회사 정보: 연혁, 비전
8. 제안 대상 브랜드: 회사명, 도메인(로고 추출용), CI 색상
```

### 자료 부족 시 처리 (A/B/C 등급)

수집한 정보의 충분도를 평가합니다:

| 등급 | 기준 | 대응 |
|------|------|------|
| **A** | 핵심 콘텐츠 충분 | 바로 생성 진행 |
| **B** | 골격은 있으나 세부 부족 | 공개 정보로 보강 가능한 부분 보강 후 사용자 확인 |
| **C** | 핵심 정보 누락 | 사용자에게 추가 자료 요청 (구체적으로 무엇이 필요한지 안내) |

### 인터뷰 방식

- 한 번에 너무 많은 질문을 하지 않는다 (최대 3개씩)
- 이미 제공된 파일(PDF, DOCX 등)이 있으면 먼저 읽고 분석한 뒤, 부족한 부분만 질문
- "~에 대한 자료가 있으신가요?" 식으로 부드럽게 질문
- 사용자가 "알아서 해줘"라고 하면 공개 정보 기반으로 최선의 판단 후 확인 요청

## Phase 1: 브랜드 분석

`scripts/ai_brand_analyzer.js`를 활용하여:

1. **로고 추출**: 제안 대상 도메인에서 로고 이미지 가져오기
2. **색상 추출**: 로고에서 dominant color 추출
3. **6:3:1 팔레트 생성**:
   - DOMINANT (60%): 브랜드 핵심색 — 배경, 간지, 넓은 면적
   - SECONDARY (30%): HSL 조화색 — 카드, 구분선, 태그
   - ACCENT (10%): 포인트 강조색 — 핵심 수치, CTA
   - 파생: DARK_BG, LIGHT_BG, CARD_BG, SUBTLE
4. **배경 이미지 생성**: 표지/간지/마무리용 그라디언트+장식 배경 (SVG→PNG)

### 색상 확인

팔레트 생성 후 사용자에게 결과를 보여주고 확인을 받는다:
```
제안 대상: [회사명]
DOMINANT: #XXXXXX (설명)
SECONDARY: #XXXXXX (설명)
ACCENT: #XXXXXX (설명)

이 색상 조합으로 진행할까요? 수정이 필요하면 말씀해주세요.
```

## Phase 2: 콘텐츠 구조화

### Step 1: 논리 구조 설계 (콘텐츠 우선)

템플릿을 먼저 고르지 않는다. 제안서의 스토리라인을 먼저 설계한다:

1. 전체 목차 구성 (챕터 나누기)
2. 각 페이지의 역할과 핵심 메시지 정의
3. 논리적 흐름 검증 (WHY → WHAT → HOW → PROOF → PROMISE)

### Step 2: 4분류 + 템플릿 매핑

논리 구조가 확정되면 각 페이지를 4분류에 배치하고, `references/02_제작시나리오_가이드.md`를 참조하여 적합한 템플릿(S01~S43)을 매핑한다.

**매핑 원칙:**
- 표지 → S01 (createCover 함수)
- 간지 → S02 (createDivider 함수) — 챕터 시작마다
- 본문-기본 → S03~S41 중 콘텐츠 유형에 맞는 것 선택
- 본문-강조 → 커스텀 또는 S 변형 (챕터당 최대 2장, 연속 불가)
- 마무리 → S43 (createEnding 함수)

### Step 3: 카피 정제

`scripts/ai_copy_refiner.js`를 활용하여:
- 표지 메인카피, 서브카피
- 각 간지의 챕터 타이틀
- 마무리 다짐 카피
- 각 페이지 헤더 메시지

를 Claude API로 정제한다. 자동 검증 루프(목표 85점+)를 돌려 일관성을 확보한다.

### 구조 확인

매핑 결과를 사용자에게 보여주고 확인을 받는다:
```
[제안서 구조]
1. 표지 — "핵심 카피"
2. 간지 Ch.1 — "챕터 제목"
3. S07 — 본문-기본: 페이지 설명
4. S12 — 본문-기본: 페이지 설명
5. S24 — 본문-강조: 페이지 설명
...
N. 마무리 — "다짐 카피"

이 구조로 진행할까요?
```

## Phase 3: PPT 코드 생성

`references/01_생성프롬프트_가이드.md`의 섹션 12 (PPT 코드 생성 가이드)를 반드시 읽고 따른다.

### 핵심 규격

**폰트 변수:**
```javascript
const FN_XB = 'Pretendard ExtraBold';
const FN_MD = 'Pretendard Medium';
const FN_TN = 'Pretendard Thin';
```

**표지 (createCover) — S01:**
```
x=0.57"  라벨 y=2.43" 18pt Medium
         메인카피 y=2.91" 40~54pt ExtraBold
         서브카피 y=4.83" 14pt Thin
         bgData 파라미터로 AI 배경 적용
```

**간지 (createDivider) — S02:**
```
x=0.57"  챕터라벨 y=2.33" 13pt Medium
         타이틀 y=3.03" 40pt ExtraBold
         보조텍스트 y=4.47" 13pt Thin
         bgData 파라미터로 AI 배경 적용
```

**마무리 (createEnding) — S43:**
```
LEFT-ALIGNED x=1.47"
         다짐카피 y=3.40" 36pt ExtraBold
         구분선 y=5.30" w=2"
         부연 y=5.63" 14pt Medium
         저작권 y=6.45" 12pt Thin
         bgData 파라미터로 AI 배경 적용
```

**본문 헤더 (모든 본문 페이지 공통 — 절대 고정):**
```
페이지번호+목차라벨: x=0.47" y=0.21" w=5.0" h=0.25"
핵심 메시지 (Bold 16pt): x=0.47" y=0.68" w=10.75" h=0.47"
부제목 (Regular 10pt): x=0.47" y=1.18" w=10.75" h=0.29"
구분선 (ACCENT): x=0.47" y=1.55" w=10.75" h=0.015"
콘텐츠 시작선: y=2.00"
콘텐츠 영역: x=0.47~11.22" (w=10.75"), y=2.00~7.50" (h=5.50")
```

**슬라이드 규격**: A4 가로 11.69 × 8.27 inches

### 코드 생성 절차

1. `pptx_vars.js` 생성 — Phase 1/2의 결과(팔레트, 카피, 배경)를 변수로 정리
2. 메인 생성 코드 작성 — pptxgenjs로 각 슬라이드 생성
3. `node generate_ppt.js` 실행하여 .pptx 출력
4. 생성된 파일 경로를 사용자에게 전달

### Fill-Rate 규칙

- 모든 슬라이드의 콘텐츠 영역 채움률 40% 이상 유지
- 빈 공간이 많으면 아이콘, 차트, 도표, 보조 텍스트 등으로 보강

## Phase 4: 시각 QA

`scripts/ai_visual_qa.js`를 활용하여:

1. 생성된 .pptx를 이미지로 변환
2. Claude Vision으로 슬라이드별 품질 분석:
   - 정렬(alignment), 채움률(fillRate), 색상(color), 타이포그래피(typography), 시각요소(visuals), 헤더(header)
3. 70점 미만 슬라이드에 대해 수정 코드 제안
4. 수정 적용 후 재생성

### QA 보고서 형식

```
[시각 QA 결과]
전체 평균: XX점
- 슬라이드 1 (표지): 85점 ✓
- 슬라이드 2 (간지): 78점 ✓
- 슬라이드 3 (본문): 62점 ✗ → 수정 필요: 채움률 부족
...

수정이 필요한 슬라이드가 N개 있습니다. 자동 수정을 진행할까요?
```

## 환경 설정

이 스킬을 처음 사용할 때 아래 사항을 확인하세요:
