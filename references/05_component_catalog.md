# 05. 컴포넌트 카탈로그 — 마스터 PPT 기반 디자인 시스템

## 개요

이 문서는 `assets/component_master.pptx`에 포함된 디자인 요소들의 공식 카탈로그입니다.
원본 파일은 사용자의 "PPT 컴포넌트 모음.pptx"이며, 총 17개 슬라이드에 걸쳐 제안서 PPT 생성에 사용되는 모든 레이아웃 컴포넌트를 정의합니다.

PPT 파서(`scripts/pptx_parser.js`)로 추출한 정확한 좌표·크기·색상 수치는 `assets/component_catalog.json`에 저장됩니다.
이 마크다운 문서는 **사람이 읽는 레퍼런스**이고, JSON은 **코드가 참조하는 데이터 소스**입니다.

---

## 파서 기반 워크플로우

```
component_master.pptx
        │
        ▼
  pptx_parser.js        ← 슬라이드별 도형·텍스트·색상·좌표 추출
        │
        ▼
  component_catalog.json ← 컴포넌트 ID별 구조화된 메타데이터
        │
        ▼
  PPT 생성 엔진          ← 콘텐츠 + 팔레트 + 컴포넌트 조합 → 최종 .pptx
```

**핵심 원칙:**
1. 마스터 PPT의 도형 좌표·크기를 **그대로** 기본값으로 사용
2. 콘텐츠 개수/길이에 따라 **변형 규칙**을 적용하여 조정
3. 샘플 색상은 클라이언트 팔레트의 **역할(role)**에 매핑하여 치환

## 변형 규칙 (공통)

### 개수 변형

콘텐츠 항목 수가 마스터 기본값과 다를 때 적용하는 규칙:

| 상황 | 처리 방식 |
|---|---|
| 항목 수 < 기본값 | 기본값 기준 레이아웃 유지, 빈 슬롯 제거 후 간격 재분배 |
| 항목 수 = 기본값 | 마스터 좌표 그대로 사용 |
| 항목 수 > 기본값 (범위 내) | 슬라이드 유효 영역 내에서 균등 분배. 폰트 1pt 축소 허용 |
| 항목 수 > 최대값 | 2장 분할 또는 상위 컴포넌트로 대체 |

### 텍스트 길이 변형

| 상황 | 처리 방식 |
|---|---|
| 텍스트 < 박스 | 그대로 유지. 정렬 규칙(좌정렬/중앙) 적용 |
| 텍스트 = 박스 | 마스터 폰트 크기 사용 |
| 텍스트 > 박스 (1단계) | 폰트 크기 최대 2pt 축소 |
| 텍스트 > 박스 (2단계) | 줄바꿈 허용, 박스 높이 확장 (하단 여백 차감) |
| 텍스트 > 박스 (3단계) | 말줄임(...) 처리 + 경고 로그 |

## 색상 치환 규칙

마스터 PPT의 샘플 색상은 클라이언트 팔레트의 **역할(role)**에 매핑됩니다.

### 역할 매핑 테이블

| 마스터 PPT 역할 | 팔레트 키 | 적용 대상 |
|---|---|---|
| 주요 강조색 (빨강/파랑 계열) | `accent-primary` | 소제목 색상, 헤더 바, 뱃지 첫 번째 색, 쉐브론 활성 |
| 보조 강조색 | `accent-secondary` | 뱃지 두 번째 색, 보조 라벨, 구분선 |
| 다크 배경 | `dark-bg` | TEXT-DARK-01 배경, 인사이트 박스 배경, 사이드바 |
| 밝은 배경 | `light-bg` | 카드 배경, 리스트 배경, 표 짝수 행 |
| 본문 텍스트 | `text-body` | 일반 본문, 설명 텍스트 |
| 밝은 텍스트 | `text-light` | 다크 배경 위 텍스트 |
| 그래디언트 시작 | `gradient-start` | 인사이트 박스, 히어로 오버레이 |
| 그래디언트 끝 | `gradient-end` | 인사이트 박스, 히어로 오버레이 |

### 뱃지/다중 색상 치환

BADGE-01처럼 여러 색상이 필요한 컴포넌트의 경우:

```
뱃지 1 → accent-primary
뱃지 2 → accent-secondary
뱃지 3 → accent-primary (밝기 +20%)
뱃지 4 → accent-secondary (밝기 +20%)
뱃지 5 → accent-tertiary 또는 neutral-mid
```

### 치환 시 주의사항

1. **Phase 1에서 추출한 색상은 반드시 공식 CI와 교차 검증** (ref: feedback_proposal_color_verify)
2. 다크 배경 위 텍스트는 명도 대비 4.5:1 이상 보장
3. 그래디언트는 시작-끝 색상 차이가 HSL 기준 H ±30도, L ±15% 이내 유지
4. 치환 후 시각 QA에서 가독성 자동 검증
