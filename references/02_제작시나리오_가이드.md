# GK Master v.0.1 — 제작 시나리오 가이드

> **목적**: 최종 수정된 GK_Master_v.0.1 템플릿(43슬라이드)의 각 페이지 구조를 분석하고,
> 어떤 콘텐츠 상황에서 어떤 슬라이드를 선택해야 하는지를 명확히 매핑합니다.
> 이 문서는 **PPT 자동 생성 프롬프트의 슬라이드 선택 근거**로 사용됩니다.

## 1. 템플릿 구조 요약

### 1.0 페이지 4분류 체계

> **모든 제안서 페이지는 4가지 유형으로 분류됩니다.**
> 이 체계는 템플릿 슬라이드 선택보다 상위 개념입니다.
> 상세 정의: `GK_생성프롬프트_가이드.md` 섹션 10 참조.

| 분류 | 템플릿 매핑 | 시각적 무게감 | 배경 | 헤더 규격 |
|------|-----------|-------------|------|----------|
| **표지** | S01 | ★★★★★ | DARK_BG / 이미지 | 자유 |
| **간지** | S02 | ★★★★ | DOMINANT | 자유 |
| **본문-기본** | S03~S41 또는 CUSTOM | ★★~★★★ | 흰색 | **고정** |
| **본문-강조** | S 변형 또는 CUSTOM | ★★★~★★★★ | 흰색 (기본) / DOMINANT 블록 / 배경이미지+오버레이 / DARK_BG 가능 | **고정** |
| **마무리** | S43 | ★★★★★ | DOMINANT / DARK_BG | 자유 |

**기본 흐름**: 표지 → (간지 → 본문 반복) → 마무리
**강조 페이지 규칙**: 챕터당 최대 2장, 연속 배치 불가

> **강조 페이지 시각 차별화**: 강조 페이지는 기본 페이지와 시각적으로 확연히 구별되어야 합니다.
> 배경색 변경, 배경 이미지 사용, 대형 다크 블록 등의 방법을 활용하여 기본 페이지와의 차이를 명확히 합니다.

#### 표지/마무리 카피라이트 규칙

> **표지(COVER)**와 **마무리(ENDING)** 페이지에는 반드시 임팩트 있는 카피라이트 문구를 포함해야 합니다.

| 구분 | 규칙 | 예시 |
|------|------|------|
| **표지 카피** | 프로젝트 제목만 넣지 말고, 제안의 핵심 가치를 선언하는 임팩트 카피 필수 | "○○의 모든 이동을 그라운드케이가 설계합니다" (O) |
| **마무리 카피** | 다짐/의지를 담은 카피 필수. 단순 인사 절대 금지 | "모든 이동을 안전하고 즐겁게 설계하겠습니다" (O) |
| **마무리 금지** | "감사합니다"만 넣는 것은 **절대 금지** | "감사합니다" (X — 절대 금지) |

#### 분류 내 레이아웃 변형 원칙

> 각 분류 내에서 여러 레이아웃 변형이 가능하나, 분류의 핵심 원칙은 반드시 유지해야 합니다.

| 분류 | 레이아웃 선택지 | 필수 유지 원칙 |
|------|---------------|---------------|
| **본문-기본** | S04~S41 중 선택 또는 CUSTOM | 흰색 배경 + 고정 헤더 필수 |
| **본문-강조** | CUSTOM 자유 설계 가능 | 고정 헤더 필수 + 시각 강도 UP |

### 1.1 템플릿 슬라이드 분류

| 구분 | 슬라이드 번호 | 용도 | 4분류 매핑 |
|------|-------------|------|-----------|
| **구조 슬라이드** | S1 | 표지 | 표지 |
| | S2 | 챕터 구분 | 간지 |
| | S3 | 목차 | 본문-기본 |
| | S42 | 서브엔딩 | 본문-강조 또는 마무리 전 강조 |
| | S43 | 엔딩 | 마무리 |
| **콘텐츠 슬라이드** | S4 ~ S41 | 본문 레이아웃 38종 | 본문-기본 또는 본문-강조 |

### 공통 레이아웃 규칙
- **슬라이드 규격**: A4 가로 (29.7 x 21.0cm / 11.69 x 8.27in)
- **색상 체계**: 제안 대상 기반 6:3:1 시스템 (상세: `GK_생성프롬프트_가이드.md` 색상 결정 프로세스)
  - DOMINANT (60%): 제안 대상 핵심 브랜드 색상 — 배경, 간지, 넓은 면적
  - SECONDARY (30%): 조화로운 보조색 — 카드, 구분선, 태그
  - ACCENT (10%): 포인트 강조색 — 핵심 수치, CTA, 하이라이트
  - 파생: DARK_BG, LIGHT_BG, CARD_BG, SUBTLE (자동 산출)
  - 기본값 (대상 불명 시): DOMINANT `#2E3092`, SECONDARY `#7B8CDE`, ACCENT `#E8B931`
- **폰트**: Pretendard 패밀리
  - **ExtraBold**: 표지 메인카피(40~54pt), 간지 타이틀(40pt), 마무리 다짐카피(36pt)
  - **SemiBold/Bold**: 본문 헤더 메시지(16pt), 카드 제목
  - **Medium**: 표지 라벨(18pt), 간지 챕터라벨(13pt), 마무리 부연(14pt)
  - **Thin**: 표지 서브카피(14pt), 간지 보조(13pt), 마무리 저작권(12pt)
  - **Regular**: 본문 텍스트, 부제목

> **색상 변수 원칙**: 이 문서 및 모든 템플릿 설명에서 언급되는 색상값(hex 코드 포함)은
> **변수**로 해석해야 합니다. DOMINANT, SECONDARY, ACCENT 등의 색상은
> 제안 대상 기업의 브랜드 색상에 따라 결정됩니다.
> 위의 기본값(`#2E3092` 등)은 대상이 불명확할 때만 적용되는 폴백(fallback)입니다.
> 각 템플릿 설명에서 "PRIMARY", "ACCENT" 등으로 참조되는 색상도 동일하게
> 제안 대상에 맞춰 동적으로 결정됩니다.

### ★ 헤더 영역 정밀 좌표 (모든 본문 슬라이드 공통 — 절대 변경 금지)

> **마스터 템플릿 실측값 기반. 커스텀 슬라이드 포함 모든 본문 페이지에 동일 적용.**

```
┌────────────────────────────────────────────────────────────────────────┐
│  페이지번호 + 목차라벨:   x=0.47in  y=0.21in  w=5.0in   h=0.25in    │
│  핵심 메시지 (Bold 16pt): x=0.47in  y=0.68in  w=10.75in h=0.47in    │
│  부제목 (Regular 10pt):   x=0.47in  y=1.18in  w=10.75in h=0.29in    │
│  구분선 (ACCENT):         x=0.47in  y=1.55in  w=10.75in h=0.015in   │
│  ~~~~~~~~ 호흡 여백 ~~~~~~~~                                          │
│  ═══════ 콘텐츠 시작선 ═══ y=2.00in ═════════════════════════════    │
│  콘텐츠 영역: x=0.47~11.22in (w=10.75in), y=2.00~7.50in (h=5.50in) │
│  하단 페이지 번호:         x=10.50in y=7.50in                        │
└────────────────────────────────────────────────────────────────────────┘
```

> **v2 변경사항**: 콘텐츠 시작선을 기존 y=1.67 → **y=2.00**으로 하향 조정 (약 0.84cm).
> 마스터 템플릿 실측 결과 실제 콘텐츠 시작이 y=1.94~2.20 범위이며,
> 전체 페이지의 시각적 밸런스가 상단으로 치우치지 않도록 보정한 값입니다.

> **시각적 풍성함 원칙**: 모든 본문 슬라이드에 최소 2가지 이상의 비텍스트 시각 요소
> (아이콘 원형, 이미지 영역, 액센트 바, 인사이트 박스, 태그, 장식 도형 등)를 포함해야 합니다.
> 상세 가이드는 `GK_생성프롬프트_가이드.md` 섹션 9.5 참조.

### ★ 시각적 밀도 원칙 (Fill-Rate)

> **콘텐츠 영역(65.9 sq.in)에서 빈 공간이 40%를 넘지 않도록 합니다.**

- 목표 Fill-Rate: **60~80%**
- 40% 미만일 경우: 아이콘, 장식 도형, 인사이트 박스, 이미지 플레이스홀더 등 추가
- 장식 요소는 대칭(symmetric) 배치 권장
- 상세 가이드: `GK_생성프롬프트_가이드.md` 섹션 11 참조

### ★ 콘텐츠 논리 우선 원칙

> **템플릿은 시각적 '그릇'이지, 콘텐츠의 논리 구조를 결정하지 않습니다.**

- 제안서의 스토리라인과 설득 논리를 먼저 확정합니다
- 확정된 논리에 맞는 템플릿을 선택합니다
- 적합한 템플릿이 없으면 **커스텀 슬라이드**를 제작합니다
- 커스텀 슬라이드도 헤더 영역 좌표와 콘텐츠 영역은 동일하게 유지합니다
- 상세 가이드: `GK_생성프롬프트_가이드.md` 섹션 10, 12 참조

## 2. 슬라이드별 상세 시나리오

### S01 — 표지 (Cover)
**구조**: 상단 라벨(18pt) → 메인 임팩트 카피(40~54pt) → 서브카피(14pt) → 하단 정보
**도형 수**: sp:4 | 효과: alpha (DOM 오버레이 30% 투명)
**색상**: DARK_BG(`#0D0D2B`) 배경 + DOMINANT 오버레이(30%투명), SECONDARY 라벨, WHITE 카피, TEXT_FAINT 서브

**마스터 실측 좌표 (pptxgenjs createCover 함수 참조)**:
| 요소 | 좌표 | 폰트 |
|------|------|------|
| 상단 라벨 ("OOO 제안서") | x=0.57" y=2.43" w=8.77" | 18pt Pretendard Medium, SEC 색상 |
| **메인 임팩트 카피** | x=0.57" y=2.91" w=9.50" h=1.92" | **40~54pt Pretendard ExtraBold**, WHITE |
| 서브 카피 | x=0.57" y=4.83" w=8.77" | 14pt Pretendard Thin, TEXT_FAINT |
| 하단 정보 | x=0.57" y=7.50" | 9pt Pretendard Thin, TEXT_FAINT |

**사용 시나리오**:
- 모든 제안서의 첫 페이지로 **필수** 사용
- **임팩트 카피 필수**: 단순 프로젝트명이 아닌, 제안의 핵심 가치를 선언하는 문구
- 카피가 1줄이면 54pt, 2줄이면 40pt로 자동 조절
- AI 모듈(ai_brand_analyzer)의 gradients.cover를 배경으로 사용 가능

**콘텐츠 매핑**:
| 요소 | 입력 소스 |
|------|----------|
| 상단 라벨 | 클라이언트명 + "제안서" (예: "Genesis VIP Shuttle 제안서") |
| 메인 임팩트 카피 | 제안 전체를 관통하는 선언형 한마디 (예: "○○의 모든 이동을\n그라운드케이가 설계합니다") |
| 서브 카피 | 핵심 메시지 1줄 부연 |
| 하단 정보 | "제안사 ○○ \| 제안 대상 ○○ \| 날짜" |

### S02 — 챕터 구분 (Section Divider)
**구조**: 챕터 라벨(13pt) → 챕터 타이틀(40pt) → 보조 설명(13pt)
**도형 수**: sp:3 | 효과: 없음
**색상**: DOMINANT 전면 배경, SECONDARY 라벨, WHITE 타이틀, TEXT_FAINT 보조

**마스터 실측 좌표 (pptxgenjs divider 함수 참조)**:
| 요소 | 좌표 | 폰트 |
|------|------|------|
| 챕터 라벨 ("CHAPTER 01") | x=0.57" y=2.33" w=8.77" | 13pt Pretendard Medium, SEC 색상 |
| **챕터 타이틀 (메시지성 카피)** | x=0.57" y=3.03" w=8.00" h=1.45" | **40pt Pretendard ExtraBold**, WHITE |
| 보조 설명 | x=0.57" y=4.47" w=8.77" | 13pt Pretendard Thin, TEXT_FAINT |

**사용 시나리오**:
- 제안서의 **대단원 구분**에 사용 (보통 3~6개 챕터)
- 챕터 전환 시 반드시 삽입
- 번호는 01부터 순차 부여
- 타이틀은 단순 챕터명이 아닌 **메시지성 카피** (예: "처음 만나는 업체의 영업이 아닙니다")

**콘텐츠 매핑**:
| 요소 | 입력 소스 |
|------|----------|
| 챕터 라벨 | "CHAPTER 01" 또는 "Module A" 등 |
| 챕터 타이틀 | 해당 챕터의 핵심 메시지 (메시지성 카피, 2줄까지) |
| 보조 설명 | 챕터에서 다루는 내용 요약 1~2줄 |

### S03 — 목차 (Table of Contents)
**구조**: 5개 항목 번호(01~05) + 카테고리 제목 + 상세 설명
**도형 수**: sp:24 | 효과: 없음
**색상**: 깔끔한 흰색 배경, DOMINANT 포인트

**사용 시나리오**:
- 표지 직후 또는 챕터 구분 직후 **필수** 배치
- 목차 항목은 5개 기준, 3~6개까지 조절 가능
- 각 항목에 카테고리명 + 1줄 설명

**콘텐츠 매핑**:
| 요소 | 입력 소스 |
|------|----------|
| 번호(01~05) | 제안서 목차 구성 |
| 카테고리 제목 | 각 챕터/섹션명 |
| 상세 설명 | 해당 섹션의 핵심 주제 1줄 |

---

### S04 — Before/After 비교
**구조**: 좌측 BEFORE(문제점) + 우측 AFTER(솔루션), 각 3개 항목
**도형 수**: sp:15 | 효과: shadow, alpha
**색상**: 대비 구조 (적색 계열 경고 vs 안정 계열 개선)

**사용 시나리오**:
- **현황 진단 → 개선 방향** 제시 시
- 기존 방식의 문제점 vs 제안 솔루션 비교
- 경쟁사 대비 자사 우위 비교
- AS-IS / TO-BE 분석

**콘텐츠 매핑**:
| 요소 | 입력 소스 |
|------|----------|
| BEFORE 제목 | 현재 과제/문제점 요약 |
| BEFORE 항목 3개 | 구체적 문제 사항 |
| AFTER 제목 | 개선된 솔루션 요약 |
| AFTER 항목 3개 | 개선 효과/방안 |

**추천 배치**: 산업 이해/과제 진단 챕터

---

### S05 — 3단 카드 (3-Column Cards)
**구조**: 3개 카테고리 카드, 각 카드에 번호 + 제목 + 설명 3줄
**도형 수**: sp:22 | 효과: 없음
**색상**: 카드 배경 F3F4F6, DOMINANT 번호

**사용 시나리오**:
- **핵심 역량 3가지** 소개
- 서비스 3대 특징/차별점
- 제안 전략의 3가지 축
- 3개 이하의 핵심 포인트를 균등 강조할 때

**콘텐츠 매핑**:
| 요소 | 입력 소스 |
|------|----------|
| 카테고리 번호(01~03) | 자동 부여 |
| 카테고리 제목 | 핵심 역량/특징명 |
| 설명 3줄 | 각 역량의 상세 내용 |

---

### S06 — 4단 카드 (4-Column Cards + Neumorphism)
**구조**: 4개 카드, 각 카드에 번호 + 제목 + 2줄 설명
**도형 수**: sp:24 | 효과: shadow, alpha (뉴모피즘)
**색상**: 뉴모피즘 쉐도우 (E2E4EF, B0B8D0)

**사용 시나리오**:
- **핵심 역량 4가지** 소개

| 요소 | 입력 소스 |
|------|----------|
| 다짐 카피 | 표지 카피의 변주 ("~~하겠습니다", "~~을 설계합니다") |
| 부연 메시지 | 서비스 라벨 또는 핵심 키워드 (예: "Shuttle · Safety · Experience") |
| 저작권 | "CONFIDENTIAL \| © 2026 회사명" |

> **표지 vs 마무리 차이**: 표지는 "선언"(x=0.57"), 마무리는 "다짐/의지"(x=1.47")
> **마무리 카피 예시**: "모든 이동을 안전하고 즐겁게 설계하겠습니다" (O) / "감사합니다" (X — 절대 금지)
