# Phase 2: 콘텐츠 구조화

## 최우선 원칙: 콘텐츠 깊이가 디자인 밀도를 결정한다

페이지가 비어 보이는 가장 큰 원인은 디자인이 아니라 **콘텐츠 부족**이다.
아이콘, 사진, 장식 도형으로 빈 공간을 채우는 것은 보조 수단일 뿐이며,
**충분하고 구체적인 내용**이 페이지를 채우는 것이 가장 좋은 디자인이다.

**따라서 Phase 3(PPT 코드 생성)에 들어가기 전에, 모든 페이지의 콘텐츠가 아래 기준을 충족하는지 반드시 확인한다.**

## Step 0: 사용자 자료 심층 분석 (Phase 2의 전제)

사용자가 제공한 분석 자료(DOCX, PDF, 텍스트 등)를 **최대한 깊이 있게** 읽고 핵심을 추출한다.

**자료 분석 체크리스트:**
1. 자료에 포함된 **모든 수치/데이터**를 추출 (시장 규모, 성장률, 비율, 금액, 인원 등)
2. 자료에 언급된 **핵심 키워드/개념**을 목록화
3. 자료의 **논리 구조**(문제->원인->해결->기대효과)를 파악
4. 자료에 **명시적으로 나오지 않지만 추론 가능한 인사이트**를 도출
5. 자료의 근거를 바탕으로 **구체적 실행 방안/전략**을 구체화

**자료가 부족할 때 콘텐츠 보강 방법 (우선순위순):**
1. **사용자 자료에서 더 깊이 추출**: 한 문장에서도 카드 2~3장 분량의 콘텐츠를 도출 가능
   - 예: "GRP 도입" -> (1) GRP의 정의 (2) 기존 상금 체계와의 차이 (3) 도입 시 기대 수익 구조 변화
2. **공개 정보 보강**: 웹 검색으로 관련 시장 데이터, 사례, 통계를 보완
3. **논리적 확장**: 자료의 방향성을 근거로 세부 전략, 실행 계획, 기대효과를 구체화
4. **비교/대조 추가**: 업계 벤치마크, 경쟁사 사례, Before/After 비교
5. **마지막 수단**: 위의 모든 방법으로도 콘텐츠가 부족하면 사용자에게 추가 자료 요청

## Step 1: 논리 구조 설계 (콘텐츠 우선)

템플릿을 먼저 고르지 않는다. 제안서의 스토리라인을 먼저 설계한다:

1. 전체 목차 구성 (챕터 나누기)
2. 각 페이지의 역할과 핵심 메시지 정의
3. 논리적 흐름 검증 (WHY -> WHAT -> HOW -> PROOF -> PROMISE)

## Step 1.5: 페이지별 콘텐츠 깊이 검증 (Step 1과 Step 2 사이 -- 필수)

각 페이지에 배치할 콘텐츠의 양과 구체성을 점검한다. **이 검증을 통과하지 못하면 Step 2로 넘어가지 않는다.**

**페이지별 최소 콘텐츠 기준:**

| 페이지 유형 | 최소 콘텐츠 |
|-----------|-----------|
| **카드 나열형** (2~4개 카드) | 각 카드에 제목(1줄) + 본문(2~4줄) + 수치 또는 근거 1개 이상 |
| **프로세스/타임라인** | 각 단계에 명칭 + 설명(1~2줄) + 기대효과 또는 산출물 |
| **KPI/수치 강조** | 수치 + 수치의 의미 설명(1줄) + 비교 기준(전년 대비, 업계 평균 등) |
| **전략/방향** | 전략 명칭 + 왜 이 전략인지(배경 1~2줄) + 구체적 실행 방안(2~3줄) |
| **비교/분석** | 비교 항목 3개 이상 x 비교 대상 2개 이상 |
| **사례/레퍼런스** | 사례 명칭 + 핵심 내용(2줄) + 시사점(1줄) |

**콘텐츠가 기준에 미달하면:**
- 해당 페이지의 콘텐츠를 보강한다 (Step 0의 보강 방법 활용)
- 보강이 불가능하면 해당 페이지를 다른 페이지와 합치거나 삭제한다
- **절대 콘텐츠가 부족한 상태로 디자인 단계에 넘기지 않는다**

**콘텐츠 깊이 예시 -- 같은 주제, 다른 깊이:**

```
[나쁜 예 -- 얕은 콘텐츠]
카드 제목: "LCK 수익 구조 전환"
카드 본문: "상금 체계 폐지 -> GRP 도입"
-> 카드 내부가 비어 보임, 디자인으로 해결 불가

[좋은 예 -- 깊은 콘텐츠]
카드 제목: "LCK 수익 구조 전환"
카드 수치: "상금 -> GRP 전환"
카드 본문:
  - 기존: 대회 상금 중심 (연 50억 규모)
  - 변화: GRP(글로벌 수익 풀) 기반 배분
  - 효과: 구단 수익 안정화, 대기업 스폰서십 고도화
  - 선례: 한진/키움증권/우리은행 참여 확정
-> 카드가 자연스럽게 꽉 참, 디자인 보조 최소화
```

## Step 2: 4분류 + 템플릿 매핑

논리 구조가 확정되면 각 페이지를 4분류에 배치하고, `references/02_제작시나리오_가이드.md`를 참조하여 적합한 템플릿(S01~S43)을 매핑한다.

**매핑 원칙:**
- 표지 -> S01 (createCover 함수)
- 간지 -> S02 (createDivider 함수) -- 챕터 시작마다
- 본문-기본 -> S03~S41 중 콘텐츠 유형에 맞는 것 선택
- 본문-강조 -> 커스텀 또는 S 변형 (챕터당 최대 2장, 연속 불가)
- 마무리 -> S43 (createEnding 함수)

### 챕터 최소 페이지 수 규칙

**간지를 만들려면 해당 챕터에 본문 페이지가 최소 2장 이상 있어야 한다.** 간지 + 본문 1장만으로 구성된 챕터는 간지의 존재 의미가 없으므로, 해당 본문 페이지를 이전 챕터에 편입하거나 강조 페이지로서 간지 없이 직접 배치한다.

```
(X) 나쁜 예:
  S16: 간지 "Ch4. 함께 만드는 캠퍼스"
  S17: 강조 페이지 "신촌과 송도, 하나의 연세"  <- 본문 1장뿐
  S18: 마무리

(O) 좋은 예:
  S16: 강조 페이지 "신촌과 송도, 하나의 연세"  <- Ch3 뒤에 바로 이어 붙임
  S17: 마무리
```

## 본문-강조 페이지 디자인 원칙 (매우 중요)

강조 페이지는 제안서의 **클라이맥스**에 해당하는 페이지다. 다른 본문 페이지들과 시각적으로 확연히 구별되어야 한다.

**강조 페이지가 배치되어야 하는 콘텐츠:**
- "우리가 이것을 하겠다" -- 핵심 제안/솔루션 선언
- "이 프로젝트는 이런 방향이다" -- 비전/방향성 제시
- 핵심 KPI/성과 수치 -- 임팩트 있는 숫자로 설득하는 장면
- 전략의 핵심 전환점 -- 스토리라인에서 가장 중요한 메시지

**강조 페이지에 부적합한 콘텐츠:**
- 레퍼런스/포트폴리오 갤러리 (이것은 본문-기본)
- 단순 데이터 테이블, 일정표
- 조직도, 팀 소개

**시각적 차별화 방법 (반드시 하나 이상 적용):**
1. **배경 색상 반전**: 흰색 배경 대신 DOMINANT 또는 DARK_BG를 전면 배경으로 사용, 텍스트를 밝은 색으로 반전
2. **대형 다크/컬러 블록**: 콘텐츠 영역 전체를 채우는 대형 배경 블록 + 밝은 텍스트
3. **배경 이미지 + 오버레이**: 관련 이미지 위에 반투명 DOM 오버레이를 깔고 흰색 텍스트
4. **대형 타이포그래피**: 핵심 수치나 메시지를 48pt+ 초대형으로 배치

강조 페이지라도 **헤더 영역(페이지번호, 핵심메시지, 부제목, 구분선)은 동일 좌표로 유지**한다. 배경이 어두울 경우 헤더 텍스트 색상만 밝게 반전한다.

## 간지(Divider) 페이지 -- 표지/마무리와의 차별화 원칙

간지는 챕터 전환을 알리는 페이지로, 표지/마무리와 배경색/디자인이 달라야 시각적 구분이 된다.

| 구분 | 배경 | 톤 |
|------|------|-----|
| **표지/마무리** | DARK_BG 또는 DOMINANT (가장 어둡고 무거운 톤) | 선언적, 임팩트 |
| **간지** | SECONDARY 또는 DOMINANT의 밝은 변형 (표지보다 밝거나 색상이 다른 톤) | 전환, 호흡 |

간지가 표지/마무리와 동일한 배경색을 사용하면 제안서의 리듬감이 사라진다. 반드시 시각적으로 구분되는 색상/패턴을 적용할 것.

### 간지 통일성 원칙 -- 한 제안서 내 간지 type은 하나로 통일

한 제안서의 모든 간지는 **동일한 type**을 사용한다. 간지는 챕터 전환을 알리는 호흡 페이지이므로 일관된 형식이 제안서의 리듬감을 만든다. 다양성은 본문 페이지에서 확보하고, 간지는 통일감을 우선한다.

**규칙:**
1. Phase 2에서 제안서 분위기에 맞는 **하나의 간지 type**을 선택한다
2. 모든 간지에 동일한 type을 적용한다
3. 챕터별 차이는 **색상 톤 변화로만** 표현한다 (예: DOM -> SEC -> DOM의 밝은 변형 -> ACC 톤)
4. 간지 type 선택 기준:

| type | 특징 | 적합한 제안서 |
|------|------|-------------|
| A (센터) | 흰 배경, 중앙 정렬, 깔끔 | 공식/정중한 제안서 |
| B (사이드바) | 좌측 다크 블록 + 우측 목차 | 정보량 많은 제안서 (하위 목차 표시 가능) |
| C (이미지분할) | 좌측 이미지 + 우측 텍스트 | 시각적 임팩트 필요한 제안서 |
| D (브랜드그래픽) | 전면 다크 + 장식 도형 | 브랜드 강조 제안서 |

**색상 톤 변화 예시 (Type B, 4챕터):**
```
Ch1 간지: 좌측 블록 DOM (#003876), 타이틀 DOM
Ch2 간지: 좌측 블록 SEC (#3D8B9E), 타이틀 SEC  
Ch3 간지: 좌측 블록 DK (#001F42), 타이틀 흰색
Ch4 간지: 좌측 블록 DOM 밝은 변형, 타이틀 DOM
```

> **이전 규칙(AP-22 간지 다양성 강제)은 폐기한다.** 4개 챕터에 A/B/C/D를 모두 쓰면 간지끼리 통일감이 없어 제안서가 4개의 다른 PPT를 이어붙인 것처럼 보이는 문제가 발생했다.

### 강조 페이지 다양성 강제 -- 한 제안서 내 highlight variant 중복 금지

마찬가지로 본문-강조 페이지가 2개 이상이면 서로 다른 variant를 사용한다.

| 강조 페이지 수 | 권장 variant 조합 |
|------------|----------------|
| 2 | dark-keywords + image-overlay-message |
| 3 | dark-keywords + image-overlay-message + split-impact |
| 4+ | 위 3종 순환, 연속 동일 variant 금지 |

같은 variant를 반복하면 "두 번째 강조"가 첫 번째와 동일해 보여 클라이맥스의 의미가 사라진다.

## Step 3: 카피 정제

`scripts/ai_copy_refiner.js`를 활용하여:
- 표지 메인카피, 서브카피
- 각 간지의 챕터 타이틀
- 마무리 다짐 카피
- 각 페이지 헤더 메시지

를 Claude API로 정제한다. 자동 검증 루프(목표 85점+)를 돌려 일관성을 확보한다.
