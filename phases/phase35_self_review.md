# Phase 3.5: 자체 검수 (PPT 생성 직후, 사용자 전달 전 -- 필수)

PPT 코드를 생성한 직후, 사용자에게 결과물을 전달하기 전에 **반드시** 아래 체크리스트를 코드 레벨에서 검수하고 문제가 있으면 수정 후 재생성한다. 이 단계를 건너뛰지 않는다.

> **검수 전 반드시 Read할 파일**: `phases/phase3_anti_patterns.md` (AP-01~AP-22 규칙 확인)

## 검수 체크리스트

**1. 요소 겹침/정렬 검사**
- 장식 도형(바, 원, 선 등)이 텍스트와 겹쳐 가독성을 해치지 않는가?
- 간지의 장식 요소가 챕터 라벨/타이틀 텍스트 영역(x=0.57~8.77)과 물리적으로 겹치지 않는가?
- 모든 텍스트 요소가 슬라이드 경계(11.69 x 8.27) 안에 있는가?

**2. 색상 대비/가독성 검사 (WCAG AA 기준)**
- 어두운 배경(DARK_BG, DOM, CB) 위의 텍스트가 밝은 색(W, ACC, TF)인가?
- 밝은 배경(W, LT) 위의 텍스트가 어두운 색(DOM, 333355)인가?
- **대비 방향 검사**: 밝은 배경 + 어두운 대면적 요소(카드, 블록)가 없는가? (AP-01 위반)
- **대비 비율 검사**: 텍스트와 배경의 명도 대비가 WCAG AA 기준(4.5:1) 이상인가?
- 다크 카드 안에 아이콘을 배치할 경우, 아이콘 배경에 밝은 색 원형을 깔아 가독성을 확보했는가?

**3. 분류별 시각 차별화 검사**
- 표지/마무리 vs 간지: 배경색이 확실히 다른가?
- 본문-강조 vs 본문-기본: 강조 페이지가 배경 반전, 대형 블록 등으로 시각적으로 확연히 다른가?
- 강조 페이지가 콘텐츠 클라이맥스에 배치되어 있는가?

**4. 폰트 규격 검사**
- 모든 본문 페이지의 핵심 메시지(hdr 함수의 msg)가 `FN_XB` (Pretendard ExtraBold)로 적용되어 있는가?
- 표지 메인카피, 간지 타이틀, 마무리 다짐카피가 모두 `FN_XB`인가?

**5. 콘텐츠 깊이 검사 (가장 중요)**
- 각 카드/블록에 제목 + 본문 2줄 이상 + 수치/근거가 포함되어 있는가?
- 콘텐츠가 부족하여 아이콘/장식 도형으로 빈 공간을 메우고 있지는 않은가?
- 빈 공간의 원인이 "디자인 문제"가 아니라 "콘텐츠 부족"은 아닌가?

**6. 카드 내부 밀도 검사**
- 개별 카드/블록 내부의 여백이 50%를 넘지 않는가?
- 카드 타이틀이 16pt 이상인가? 카드 본문이 12pt 이상인가?
- 슬라이드 전체 콘텐츠 영역 채움률이 60% 이상인가?

**7. Content Box 준수 검사**
- 모든 본문 페이지의 콘텐츠가 Content Box 경계(x=0.53~11.16", y=1.77~7.68") 안에 배치되어 있는가?
- Content Box 내부 패딩(0.15")이 지켜지고 있는가?

**8. 헤더 일관성 검사**
- 모든 본문 페이지(기본+강조)의 헤더 좌표가 동일한가? (x=0.47, y=0.21/0.68/1.18/1.55)
- 강조 페이지에서도 헤더가 생략되지 않았는가?

**9. 안티패턴 위반 검사 (AP-01~AP-22)**
- [AP-01] 본문 페이지 콘텐츠 영역 내 단일 다크 블록 면적이 25%를 초과하는가?
- [AP-02] 상하 하드 컬러 분할(50:50)이 있는가?
- [AP-03] 내부가 빈 장식 도형이 있는가?
- [AP-04] 배너/히어로 영역에 텍스트만 있고 시각 요소가 없는가?
- [AP-05] 텍스트박스 width/height 부족으로 텍스트가 도형 밖으로 빠져나가거나 강제 줄바꿈되는 곳이 있는가?
- [AP-06] 카드 그리드 내 카드들의 폰트 크기가 모두 동일한가?
- [AP-07] 카드 그리드 내 헤드라인/구분선/본문의 좌측 시작 x좌표가 모두 같은가?
- [AP-08] ACC 색상 도형의 line 속성이 모든 슬라이드에서 동일한가?
- [AP-09] 배경 PNG 파일이 진짜 PNG 바이너리(매직 89 50 4E 47)인가? pngToDataURI는 `image/png;base64,` 접두어(no `data:`)를 사용하는가?
- [AP-10] 본문 페이지에서 cardGrid({ style: 'kpi' }) 다크 변형을 쓰지 않는가?
- [AP-11] cover.js / ending.js / divider.js에 placeholder 도형이 남아있지 않은가?
- [AP-12] 한글 텍스트박스에 charSpacing 옵션이 들어가 있지 않은가?
- [AP-13] 가로 카드 그리드의 카드 수가 4 이하인가? 카드 폭이 2.20" 이상인가?
- [AP-14] generate_ppt.js에 이모지가 포함되어 있지 않은가?
- [AP-15] `bold: true` 가 generate_ppt.js와 components에 0건인가?
- [AP-16] 본문 페이지의 Content Box 수직 점유율이 85% 이상인가? 한 컴포넌트만 배치된 본문 페이지 금지
- [AP-17] 아이콘 자리에 'X'/'!'/'?'/빈 원이 없는가?
- [AP-18] 본문 페이지에 최소 2종 시각 구성 요소가 결합되어 있는가?
- [AP-19] 카드 본문이 3줄 이하인가? 4줄 초과 시 bullets로 분해되어 있는가?
- [AP-20] divider Type C 사용 시 bgImage 경로 검증 로직이 있는가?
- [AP-21] 카드 상단에 얇은 색 바(accent bar)가 없는가?
- [AP-22] 연속 3장 이상 동일 시각 구조를 사용하지 않았는가? LR 비율 40% 이상인가?

**10. 여백 정량 기준 검사**
- 슬라이드 Fill Rate가 65% 이상인가?
- 연속 빈 영역(Empty Patch)이 8 in2 이하인가?
- 각 카드의 Card Fill Rate가 50% 이상인가?
- 위반 시: 콘텐츠 보강 -> 폰트 확대 -> 도식화 -> 이미지 -> 카드 수 축소 순으로 대응

**11. 간지 통일성 / 강조 페이지 다양성 검사**
- 한 제안서 내 모든 간지가 **동일한 type**을 사용하는가?
- 간지 간 차이가 색상 톤 변화로만 표현되고 있는가?
- 간지를 가진 챕터에 본문 페이지가 **최소 2장 이상** 있는가? (1장뿐이면 간지 삭제)
- 강조 페이지가 2개 이상이면 서로 다른 variant를 사용하는가?

**12. Insight Box / 마무리 페이지 검사**
- Insight Box가 모든 본문 페이지에 의무 배치되어 있지 않은가? (적재적소 원칙)
- Insight Box 폰트가 14pt 이상인가? 높이가 0.60" 이상인가?
- 마무리 페이지의 히어로가 약속/다짐 카피인가? ("감사합니다"가 히어로가 아닌가?)
- 마무리 히어로 텍스트박스 width가 7" 이상인가?

## 검수 후 조치

- 문제가 발견되면 코드를 수정하고 재생성한다
- 수정 내용을 간략히 기록하고, 수정 완료된 파일을 사용자에게 전달한다
- 검수 통과 시에만 결과물을 전달한다
