# Phase 1: 브랜드 분석

`scripts/ai_brand_analyzer.js`를 활용하여:

1. **로고 추출**: 제안 대상 도메인에서 로고 이미지 가져오기
2. **색상 추출**: 로고에서 dominant color 추출
3. **6:3:1 팔레트 생성 (Adobe Color Harmony 기반)**:
   - DOMINANT (60%): 브랜드 CI 색상 그대로 사용 — 배경, 간지, 넓은 면적
   - SECONDARY (30%): **DOM과 독립적인 조화색** — 카드, 구분선, 태그
   - ACCENT (10%): **DOM과 독립적인 강조색** — 핵심 수치, CTA
   - 파생: DARK_BG, LIGHT_BG, CARD_BG, SUBTLE (DOM 기반으로 통일감 유지)
4. **배경 이미지 생성**: 표지/간지/마무리용 그라디언트+장식 배경 (SVG->PNG)

## 색상 조화 규칙 (매우 중요)

SEC/ACC는 브랜드 색상의 밝기/채도 변형이 **아닌**, 색상환(Color Wheel)에서 조화 규칙에 따라 선택된 **독립적인 색상**이다.

| 규칙 | 색상환 각도 | 느낌 | 적합한 제안서 |
|------|-----------|------|-------------|
| **Split-Complementary** (기본값) | SEC: +150deg, ACC: +210deg | 세련된 대비 | 일반 비즈니스 |
| **Triadic** | SEC: +120deg, ACC: +240deg | 역동적, 활기찬 | 스포츠, 이벤트 |
| **Complementary** | SEC: +180deg, ACC: +150deg | 강한 대비, 주목도 | KPI 강조, 성과 |
| **Analogous + Complement** | SEC: +30deg, ACC: +180deg | 안정감 + 포인트 | 공식 제안서 |
| **Square** | SEC: +90deg, ACC: +270deg | 다채롭고 풍부한 | 문화 행사, 크리에이티브 |

Phase 0에서 제안서의 주제/분위기를 파악한 뒤, `mood` 파라미터로 적절한 조화 규칙을 자동 선택한다. 사용자가 수동으로 지정할 수도 있다.

## 색상 검증 (Phase 1 필수 단계)

팔레트 생성 후, 사용자에게 보여주기 전에 **반드시 아래 검증을 수행**한다:

**1. 소스 색상 정확성 검증 (가장 중요)**
- 로고에서 추출한 `_source` 색상이 해당 브랜드의 **공식 CI 색상**과 일치하는지 웹 검색으로 교차 확인한다
- 검색어: "[브랜드명] CI 색상", "[브랜드명] brand color", "[브랜드명] corporate identity"
- 공식 CI 가이드가 있으면 Pantone 코드 -> Hex 변환으로 정확한 값을 확인한다
- **로고 추출값과 공식 CI값의 차이가 Hex 기준 각 채널 +/-16 이상이면 공식 CI값을 우선 적용**한다

> **왜 이 검증이 필수인가**: 로고 이미지에서 색상을 추출하면 이미지 압축, 그라데이션, 투명도 등으로 인해 실제 CI 색상과 다른 값이 추출될 수 있다. 실제로 아산병원 프로젝트에서 CI 색상(#006272 틸)이 아닌 #003876(네이비)이 추출되어, 전혀 다른 브랜드인 연세대학교와 거의 동일한 팔레트가 생성된 사례가 있다.

**2. 이전 프로젝트와의 색상 유사도 체크**
- 작업 디렉토리에 이전 프로젝트의 palette.json이 있으면 DOM 색상을 비교한다
- 두 프로젝트의 DOM이 Hex 기준 각 채널 +/-32 이내로 유사하면, harmony rule을 다르게 적용하여 SEC/ACC를 차별화한다

## 색상 확인

검증을 통과한 팔레트를 사용자에게 보여주고 확인을 받는다:
```
제안 대상: [회사명]
공식 CI 색상: #XXXXXX (출처: [공식 CI 가이드/웹사이트])
DOMINANT: #XXXXXX (설명)
SECONDARY: #XXXXXX (설명)
ACCENT: #XXXXXX (설명)

이 색상 조합으로 진행할까요? 수정이 필요하면 말씀해주세요.
```
